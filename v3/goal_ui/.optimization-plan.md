# goal_ui RuVector WASM Optimization Plan

**Branch:** `feat/goal_ui-ruvector-wasm`
**App:** `v3/goal_ui/` (`@ruflo/research` v0.1.0, Vite/React/TS, port 8080)
**Goal:** Optimize for ruvector WASM + AgentDB primitives, validate every UI element + workflow via Playwright, brand consistently as RuFlo.

## Resumption Protocol (read this every loop iteration)

1. `cd /Users/cohen/Projects/ruflo` (always use absolute paths; loop cwd resets)
2. Open this file. Find the FIRST unchecked `- [ ]` in "Sequenced Steps".
3. Read its "DoD" (Definition of Done) and "Artifact" lines.
4. Execute ONLY that step. If it requires a sub-swarm, use Task tool with named agents + SendMessage.
5. **If the step modifies any code under `src/`, `index.html`, `vite.config.ts`, `package.json`, or `functions/` — RUN THE LOCAL BROWSER VALIDATION GATE** (see "Browser Validation Gate" below) before marking done. No exceptions for "small" UI changes.
6. Mark the step `- [x]` in this file. Append a one-line note under "Loop Log" with timestamp + commit SHA + browser validation result.
7. Stage + commit with message `feat(goal_ui): step NN — <title>` on `feat/goal_ui-ruvector-wasm`.

9. Every 5 completed steps, run an "Honesty Checkpoint" (see below) before proceeding.
10. Stop after one step per iteration unless the step explicitly says "fast-follow".

## Browser Validation Gate (MANDATORY for any code-modifying step)

**Hard requirement from user — any step that changes runtime behavior MUST be validated end-to-end in a real browser before being marked `[x]`. This applies to UI changes, env-var changes, dep changes, Vite config changes, and function handler changes.**

Procedure:

1. `cd /Users/cohen/Projects/ruflo/v3/goal_ui && npm install` (if `package.json`/`package-lock.json` touched)
2. `cd /Users/cohen/Projects/ruflo/v3/goal_ui && npm run dev` in background. Capture the dev URL (default `http://localhost:8080`).
3. Use the **agent browser** (Playwright via `ruflo-browser:browser-test` skill, or Playwright directly once Step 13 lands) to:
   - Navigate to every route the step affects (`/`, `/demo`, `/agents`, plus `/widget-test.html` if widget changed)
   - Exercise the workflow the step touched (clicks, form submits — happy path)
   - Capture **all browser console messages** for the duration of the visit
4. **Pass criteria** — ALL of the following must hold:
   - HTTP 200 on every route visited
   - **Zero `console.error(...)` messages**
   - **Zero unhandled promise rejections**
   - **Zero React error-boundary triggers**
   - Build does not regress: `npm run build` (and `npm run build:widget` if widget code changed) returns exit 0
   - Any new feature flag the step introduced is visible/functional with the flag both ON and OFF
5. **Fail handling** — if any pass criterion fails: do NOT mark the step `[x]`. Mark `- [!]` instead, append the console errors verbatim under Loop Log as `BLOCKED: console error <full message>`, and stop the iteration (don't move to next step). Next loop fire either fixes the regression or escalates.
6. Append to Loop Log: `browser-validated: <route list> · 0 errors · build pass`

**Pure-doc steps (Phase 0 inventory steps, plan edits)** — exempt from the gate, but you MUST state "no runtime change — browser gate skipped" in the Loop Log entry.

### Phase 5 — Security & Branding

- [ ] **Step 22a — Security: API key isolation.** Audit the bundle for any string starting with `sk-`, `sk-ant-`, `AIza`, etc. Add a pre-commit grep + a CI step that fails on these patterns in `dist/` or `src/`. Ensure `ANTHROPIC_API_KEY` is **never** prefixed `VITE_*`. Document client vs server env vars in `example.env`. DoD: grep returns zero matches; CI rule landed.
- [ ] **Step 22b — Security: function CORS + rate limit + token.** In `functions/server.ts` and each GCF handler: lock CORS to `https://goal.ruv.io` (prod) + `http://localhost:8080` (dev), reject other origins; add per-IP token bucket (60 req/min, 429 on overflow); validate `X-RuFlo-Token` against `RUFLO_FUNCTIONS_TOKEN` env var. The same token is set via `VITE_FUNCTIONS_PUBLIC_TOKEN` in the bundle (this is a weak control; rate-limit is the real defense). DoD: requests without token → 401, with valid token from disallowed origin → 403, valid+allowed at >60/min → 429.
- [ ] **Step 22c — Security: prompt injection defense.** Wrap every user-supplied string sent to Anthropic in `<user_input>...</user_input>` delimiters in handler templates. Validate every LLM response with Zod schemas before passing to UI. Generated text is NEVER eval'd, NEVER passed to `dangerouslySetInnerHTML`, NEVER concatenated into other prompts unwrapped. DoD: every handler has a Zod schema; one negative test (intentionally malformed LLM response) routes to the safe-default branch.
- [ ] **Step 22d — Security: CSP + supply chain.** Add `<meta http-equiv="Content-Security-Policy" ...>` to `index.html` per ADR-093 §S4. Pin ruvector + ONNX deps to exact versions in `package.json` (no `^`). Add `npm audit --production --audit-level=high` to CI. Verify ONNX model SHA at load time (ruvector flag). DoD: CSP header present in built `dist/index.html`, all wasm pkgs pinned, audit passes.
- [ ] **Step 22e — Security: RVF format hardening.** Browser RVF deserializer rejects: mismatched magic, version > MAX_SUPPORTED, per-entry size > 256KB, vector dim mismatch. Add 5 negative-test cases (`tests/unit/rvf-format.spec.ts`). DoD: all 5 negative cases throw with descriptive errors; happy-path test still passes.
- [ ] **Step 23 — Terminology + branding audit.** Grep for "Lovable", inconsistent capitalization of "RuFlo"/"Ruflo", legacy product names. Verify favicon, page title, meta tags, footer all say "RuFlo Research". Update `index.html` + `public/` accordingly. DoD: zero hits for forbidden terms; Playwright asserts `<title>` + meta og:site_name.
- [ ] **Step 24 — Widget verification.** `npm run build:widget` → embed widget in `public/widget-test.html` → Playwright loads it → asserts widget renders + RVF flag respected + CSP doesn't break it. DoD: widget e2e test passes.

## Risks & Open Questions

- **R1 — `.env` access.** Loop currently can't read repo `.env`; non-secret config falls back to `example.env`. Block only if a step needs a real API key.
- **R2 — WASM bundle size.** ONNX MiniLM ~25MB. Mitigation: lazy-load behind dynamic import, only when flag on.
- **R3 — Widget compatibility.** ruvector WASM may not load inside cross-origin widget iframe. Step 24 verifies; fallback is widget keeps Supabase-only path.
- **R4 — Supabase coupling.** Auth/sessions remain on Supabase — don't migrate those. Migration matrix (Step 04) is the source of truth.
- **R5 — Loop drift.** If a loop iteration completes a step partially, mark `- [!]` not `- [x]`. Next iteration re-reads DoD before claiming done.
- **A1 — Assumption:** ruvector packages already in node_modules at repo root resolve from `v3/goal_ui/` via npm workspaces or hoisting. Step 05 verifies; if not, add to goal_ui's own package.json.
- **A2 — Assumption:** Playwright chromium install is allowed. Step 13 confirms.
